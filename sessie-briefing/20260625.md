## SAC-wandelschaal (moeilijkheidsberekening)

**Score = afstand(km) + stijging(/100m) + weerfactoren**

| Factor | Punten |
|--------|--------|
| Per km afstand | +1 |
| Per 100m stijging | +1 |
| Temperatuur ≥ 25°C | +2 |
| Neerslag ≥ 5mm | +2 |
| Wind ≥ 30 km/u | +1 |

| Score | Code | Label |
|-------|------|-------|
| 0–5 | T1 | Wandelen |
| 6–10 | T2 | Bergwandeling |
| 11–16 | T3 | Veeleisende bergwandeling |
| 17–22 | T4 | Alpine wandeling |
| 23–28 | T5 | Veeleisende alpine wandeling |
| 29+ | T6 | Moeilijke alpine wandeling |

## Prioriteiten sessie 06

### 1. GPX route tekenen op kaart in route.html (T2-003)

**Probleem:** `routes/route.html` toont Leaflet kaart op startpunt maar tekent de route niet. trackPoints worden opgeslagen in de creator JSON maar niet doorgegeven aan de route detail pagina.

**Oplossing:** trackPoints opslaan in de route JSON export (max 500 punten, gesamplet) en inladen in `route.js` om de route als polyline te tekenen.

**Bestanden:** `js/route.js`, `js/creator.js`

---

### 2. Route kaartpagina (T1-007)

**Doel:** `routes/[id]-map.html` — aparte pagina met fullscreen interactieve Leaflet kaart + GPX route overlay. Opent in apart tabblad via "Route openen" knop op route detail pagina.

**Onderdelen:**
- Fullscreen Leaflet kaart
- GPX route als polyline
- Startpunt marker
- Terug-knop naar route detail
- Topbar + footer

---

### 3. Draft management (T1-008)

**Doel:** Routes met `status: "draft"` tonen een "Binnenkort" badge op de grid maar zijn niet klikbaar. Gepubliceerde routes zijn klikbaar.

**Bestanden:** `js/home.js`, `js/wandelingen.js`, `css/home.css`, `css/wandelingen.css`

---

### 4. Print CSS voor routepagina (T7-001)

**Doel:** `@media print` in `route.css`:
- A4 formaat
- Hero foto behouden
- Interactieve kaart → verbergen of statisch beeld
- Alle links actief en zichtbaar
- Topbar en footer verbergen
- Galerij netjes op pagina

---

### 5. Filters op wandelingen.html (T4-001) — optioneel

**Doel:** Filter routes op moeilijkheid (T1-T6) en regio. Eenvoudige checkbox of dropdown filters boven het grid.

## Architectuurafspraken (bevestigd sessie 05)

| Onderwerp | Afspraak |
|-----------|---------|
| **Script volgorde** | Eruda → Supabase SDK → i18next CDN → i18n.js → auth.js → topbar-auth.js → analytics.js → app.js → Leaflet (indien nodig) → [pagina].js |
| **Eruda** | Niet verwijderen als aanwezig in HTML pagina |
| **CSS** | Altijd in `<head>` |
| **Favicon** | Elke nieuwe pagina krijgt favicon tags na `<title>`, vóór CSS |
| **Cache busting** | `?v=x.x.x` aan script tags bij elke deploy |
| **Cloudinary** | Hero w_1200,f_auto · Galerij w_800,f_auto · Thumbnail w_400,f_auto |
| **Route bestanden** | Volledige JSON in `routes/[id].json` · Overzicht in `routes/routes.json` |
| **Route IDs** | Altijd lowercase met koppeltekens, geen spaties |
| **Leaflet CDN** | `https://cdn.jsdelivr.net/npm/leaflet@1.9.4/dist/leaflet.min.js` (geen integrity check) |
| **i18n init** | Centraal in app.js — paginascripts roepen geen init() aan |
| **Rollen** | gast / creator / admin via `role` kolom in profiles tabel |
